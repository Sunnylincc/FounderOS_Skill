# CLAUDE.md — FounderOS repository instructions

## Mission
Maintain and improve FounderOS as a practical, vendor-neutral operating toolkit for AI-native startup execution.

## Content standards
- Write original, concise, actionable Markdown.
- Avoid platform lock-in; include Claude, Codex, Cursor, and other agents where relevant.
- Prioritize artifacts founders can use immediately.
- Every stage output should include assumptions, risks, and quality gates.

## Change workflow
1. Identify target stage (Idea/MVP/Launch/Scale).
2. Update related prompt(s), template(s), and workflow(s) together.
3. Ensure examples reflect current best practice.
4. Keep README structure and quickstarts accurate.

## Quality bar
- No motivational fluff.
- No unverifiable claims.
- No legal/financial/security advice framing.
- Include devil's advocate sections for strategic decisions.

## Style
- Use direct headings and bullets.
- Prefer checklists and tables for execution.
- Keep prompts deterministic and input-driven.
