# CODEX.md — FounderOS repository instructions

## Objective
Keep this repository a high-signal prompt + template toolkit that helps founders run Idea → MVP → Launch → Scale workflows with coding agents.

## Agent-neutral constraints
- Do not assume a single AI vendor or IDE.
- Keep instructions executable in Claude, Codex, Cursor, and similar tools.
- Use clear inputs, expected outputs, and quality checks.

## Editing policy
- When changing a prompt, verify corresponding template/workflow alignment.
- Keep examples realistic, specific, and falsifiable.
- Favor operational clarity over theoretical completeness.

## Required output characteristics
- Concrete Markdown artifacts.
- Explicit assumptions and unknowns.
- Risk flags and mitigation options.
- Devil’s advocate counterargument for major bets.

## Safety constraints
Always remind users to do human review for production systems, especially where user data, billing, regulated domains, or security-sensitive workflows are involved.
